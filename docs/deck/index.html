The Problem

Security teams rebuild the same automation, enrichment, and reporting logic around every new vendor API. The result is fragile, non-portable workflows that break with each contract renewal.

Integration churn

Each new platform demands bespoke glue code that nobody else can maintain.

Stranded intel

Indicators live in silos — enrichment, context, and attribution can't travel with them.

Workflow lock-in

Automations are tied to specific tools, so every vendor change restarts the clock.

What GNAT Does

Normalize

159 connectors behind one unified client surface — same call signature everywhere.

Translate

STIX 2.1 as the stable contract. Objects, bundles, and patterns that travel between tools.

Analyze

Diamond Model attribution, kill-chain mapping, campaign tracking, and hypothesis evaluation.

Anchor the ecosystem

Hub platform that SandGNAT, SenseGNAT, RedGNAT, and GNAT-gui plug into.

Core Architecture

Data layer
  • STIX 2.1 ORM — pure Python property bag, no SQLAlchemy
  • urllib3 (sync) + httpx (async) — zero requests dependency
  • INI-based config — zero-dep, per-platform sections
  • Optional Rust extension (gnat._core) for hot paths
Pipeline layer
  • Ingest: 15 readers · 13 mappers · Kafka telemetry · Redis dedup
  • Export: EDL · Netskope CE delivery targets
  • Schedule: croniter-based FeedJob / FeedScheduler
  • Search: Solr 9.x full-text sidecar with ORM integration
  • Serve: FastAPI + TAXII 2.1 endpoint

AI & Analysis

LLMClient

Unified facade — Claude, OpenAI, Grok (xAI), Gemini. Automatic fallback. urllib3-based, no requests.

Agents

ResearchAgent (SourceReader), ParsingAgent (RecordMapper), quality agents, repo-maintenance agents, secrets hygiene.

Attribution

Diamond Model, kill-chain mapping, campaign auto-link, infrastructure classification, hypothesis engine (Hy + YAML + Prolog rule backends).

Detection rules

STIX → Sigma, YARA, Suricata, Snort. Hunt packages, ATT&CK coverage, validation. (huntgnat plugin)

Reports

AI-assisted PDF + DOCX via ReportLab / python-docx. ResearchLibrary + CurationJob pipeline.

NLP

Built-in IOC extraction + Claude backend. Extracts indicators from freeform text at analyst speed.

Getting Started

# Install
pip install gnat
pip install "gnat[all]"        # everything

# Configure
cp config/config.ini.example ~/.gnat/config.ini

# Use the CLI
gnat ingest --source taxii --output stix
gnat report --template executive --format pdf
gnat tui                        # launch TUI

Python 3.9–3.13 · Apache 2.0 · INI config, no secrets in code
